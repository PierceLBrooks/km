Concept: Be-Held

Superconcept: Be-Obstructed

Definition: An object in a Be-Held state can only be Moved by the agent of this state. An object is put in this state by Hold. An object is taken out of this state by Let-Go-Of.

Required Relationships:

object: the thing that is held. It cannot be Moved unless done by the agent
agent: the thing that is ing the object. Only it can move the object

Examples:

  1. John is holding the book.
    agent: John
    object: the book
    2. Comment: Here the book is in a Be-Held state, since no one else can move it, but John.

Neighboring Concepts:

Similar to...But different because...
Be-Obstructed Be-Obstructed is more general, preventing all movements.
Be-Confined Be-Confined restricts the movement of the object out of a certain enclosure.
Be-Contained An object is contained in a container. It can Move-Out of it at any time.
Be-Restrained The object of a Be-Restrained state is unable to Move.
Be-Shut-Out The object of a Be-Shut-Out is unable to Move-Into an enclosure.
Be-Supported The object of a Be-Supported is prevented from Falling.

Comment: Unlike the other Be-Obstructed states, Be-Held has an agent. Any Move of an object in a Be-Held state must be done by the agent of the Be-Held and only this agent can Let-Go-Of the object.

Let-Go-Of takes an object out of a Be-Held state. Hold puts the object into the Be-Held state.

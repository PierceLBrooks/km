Concept: Close

Superconcept: Block

Definition: Close a Portal in a Barrier so that things cannot Move-Through it.

Required Relationships:

object: The Portal that is closed. This action changes the state of the object, from Be-Open to Be-Closed.

Other Common Relationships:

agent: the entity that causes this action
instrument: the entity that takes on the role of Portal-Covering as a result of this action.
Comment: The slots used to describe Make-Inaccessible actions in general can be used here, too. For example, destination and by-means-of.

Examples:

  1. John closed the door.
    object: the doorway
    agent: John
    instrument: the door
    2. Comment: The doorway ceases to be a Portal, and the door becomes a Portal-Covering.
  2. John closed the window with a gentle nudge.
    agent: John
    object: the space inside the window frame
    by-means-of: a gentle nudge
    3. Comment: The space inside the window frame ceases to be a Portal, and the window (the glass itself) becomes a Portal-Covering.

Neighboring Concepts:

Similar to...But different because...
Make-Inaccessible Make-Inaccessible is more general than Close, in that Close applies only to portals.
Block Block makes a path inaccessible to things moving through it or along it.
Obstruct Obstruct prevents an object from moving.

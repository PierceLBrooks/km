Concept: Be-Stored

Superconcept: State

Definition: The state of an object in which it's stored in a base. The Store action puts objects into this state, and the Withdraw action takes them out of the state.

Required Relationships:

object: the entity that's stored
base: the Tangible-Entity in which the object is stored

Examples:

  1. The cookies were stored in the jar.
    object: the cookies
    base: the jar

Neighboring Concepts:

Similar to...But different because...
Be-Contained Be-Stored is a special case of Be-Contained in which the object is put away for a longer period of time, usually for safe keeping.
Store The Store action puts objects into the Be-Stored state
Withdraw The Withdraw action takes objects out of the Be-Stored state

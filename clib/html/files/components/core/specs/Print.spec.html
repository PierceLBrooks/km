Concept: Print

Superconcept: Mark

Component Semantics:

Print
Definition: to render a written Message in physical form; in particular, to take a Document (a kind of Message) and embody it as a Physical-Document.

Required Relationships:

object: The Document being embodied (in ink etc.).
result: The Physical-Document that is the result of the Printing.

Neighboring Concepts:

Similar to...But different because...
Mark Print is a more specific type of Mark in which the result is, by definition, a Physical-Document. With Mark, a Physical-Document *may* be the result, or it may be something less document-like (e.g. spray paint on a subway car); with Print the result *must* be a Physical-Document.
Write The object of (or input to) a Write is an instance of Information whereas the object of a is a Document (a kind of Message). Also, a results in an intangible Message, specifically a Document whereas results in a Physical-Document.

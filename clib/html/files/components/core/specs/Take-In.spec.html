Concept: Take-In

Superconcept: Move-Into

Definition: Take-In is a special type of Move-Into in which a Tangible-Entity, playing the role of a Container, takes something from the outside into oneself.

Required Relationships:

object: the thing that is taken in
agent: the Tangible-Entity, playing the role of a Container, that takes in the object. (Note: the agent is also the base of the Take-In.)

Other Common Relationships:

path: the Portal through which the object is taken in
origin: the initial location of the object before the Take-In. This must be a Spatial-Entity that is-outside the agent
destination: the final location of the object after the Take-In. This must be a Spatial-Entity that is-inside the agent
toward: the location toward which the object moves during the Take-In. Unlike the destination, the object need not reach this place, but it may
away-from: the location that the object moves away from during the Take-In. Unlike the origin, the object need not start at this place, but it may

Examples:

  1. John takes in oxygen through his nose.
    object: oxygen
    agent: John
    path: John's nose

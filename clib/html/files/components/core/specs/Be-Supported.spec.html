Concept: Be-Supported

Superconcept: Be-Obstructed

Definition: The object of a Be-Supported is prevented from Falling. An object is put in this state by Obstruct. An object is taken out of this state by Unobstruct.

Required Relationships:

object: the thing that is prevented from Falling.

Other Common Relationships:

instrument: the thing being used to support the object

Examples:

  1. The beam holds up the roof.
    object: the roof
    instrument: the beam
    Comment: The roof cannot fall down because it is supported by the beam.

Neighboring Concepts:

Similar to...But different because...
Be-Obstructed Be-Obstructed is more general, preventing all movements.
Be-Confined Be-Confined restricts the movement of the object out of a certain enclosure.
Be-Contained An object is contained in a container. It can Move-Out of it at any time.
Be-Restrained The object of a Be-Restrained state is unable to Move.
Be-Shut-Out The object of a Be-Shut-Out is unable to Move-Into an enclosure.
Be-Held has an agent that is allowed to Move the object.

Comment: An object can be put in a Be-Supported state by Support action, and can be taken out of such a state by a Let-Fall. While in a Be-Supported state, the object cannot Fall.

Concept: Interpret

Superconcept: Action

Definition: An agent interprets a message, gleaning Information from it. The agent comes to know that information as a result.
Comment: Interpret is the third and final step of Communicate. For perspective, here's the overall model of communication. Information is first converted into a message by the Express subevent, then the message is moved to the recipient by the Convey subevent, and finally the message is converted back into information by this subevent.

Required Relationships:

agent: the Tangible-Entity that receives and interprets the message
object: the Message that is interpreted
Comment: As a result of this action, the result becomes the Information that was encoded in the message, and the agent comes to know that information (i.e. the information becomes the object of a Be-Known state in which the experiencer is the agent of this interpret action.

Examples:

  1. Tom interpreted the message.
    agent: Tom
    object: the message
    2. Comment: Tom comes to know the information encoded in the message.

    Neighboring Concepts:

    Similar to...But different because...
    Learn Learn is a sub type of Interpret in which the information encoded in the message is new to the agent interpreting it
    Forget Forget is the opposite of Interpret; it causes an entity to cease knowing some information

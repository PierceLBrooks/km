Concept: Shut-Out

Superconcept: Obstruct

Definition: Prevent an object from moving into an enclosure. This action results in the object being in a Be-Shut-Out state, which can be undone by Admitting the object

Required Relationships:

object: The thing that's Shut-Out. The object is put in a Be-Shut-Out state by this action.
base: The Tangible-Entity, playing the role of Container, from which the object is Shut-Out

Other Common Relationships:

agent: the entity that causes the object to be shut out
location: the place inside the enclosure where the object is prevented from going

Examples:

  1. After losing his key, Tom couldn't get into his house.
    object: Tom
    base: house
    caused-by: losing key

Neighboring Concepts:

Similar to...But different because...
Block Prevent things from moving through some place, or along some object, by blocking such movements.
Confine Confine is more specific than Obstruct. It prevents the object from moving out of its enclosure.
Hold Hold is more specific than Obstruct. The object that's held can only be moved by the thing holding it.
Restrain Restrain is more specific than Obstruct. It prevents the object from moving at all.
Support Support is more specific than Obstruct. It prevents the object from falling.

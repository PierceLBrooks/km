Concept: Be-Confined

Superconcept: Be-Obstructed

Definition: The object of a Be-Confined is unable to Move-Out-Of the base (i.e. an enclosure). An object is put in this state by Confine. An object is taken out of this state by Release.

Required Relationships:

object: the thing that cannot Move-Out-Of.
base: the thing inside which the object is confined.

Examples:

  1. The illegal immigrants were held at a detention center.
    object: the illegal immigrants
    base: the detention center
    2. Comment: Here "held" is translated to "confined".
  2. The boy was confined to his room by his parents.
    object: the boy
    base: his room

Neighboring Concepts:

Similar to...But different because...
Be-Obstructed Be-Obstructed is more general, preventing all movements.
Be-Contained An object is contained in a container. It can Move-Out of it at any time.
Be-Restrained The object of a Be-Restrained state is unable to Move.
Be-Shut-Out The object of a Be-Shut-Out is unable to Move-Into an enclosure.
Be-Supported The object of a Be-Supported is prevented from Falling.
Be-Held has an agent that is allowed to Move the object.

Comment: An object can be put into a Be-Confined state by a Confine action, and can be taken out of such a state by a Release action. While in this state, the object cannot Move-Out-Of the base.

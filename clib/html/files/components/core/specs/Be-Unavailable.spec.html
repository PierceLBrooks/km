Concept: Be-Unavailable

Superconcept: State

Component Semantics:

Be-Unavailable
Definition: the state (of a Resource w.r.t. the original base) resulting from a Release-Resource.

Required Relationships:

object: the Entity playing the role of a Resource that becomes unavailable.

Other Common Relationships:

base: w.r.t. which the Resource was released.

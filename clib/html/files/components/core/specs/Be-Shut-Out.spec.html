Concept: Be-Shut-Out

Superconcept: Be-Obstructed

Definition: The object of a Be-Shut-Out state is unable to Move-Into an enclosure (the base). An object is put in this state by Obstruct. An object is taken out of this state by Unobstruct.

Required Relationships:

object: the thing that is unable to Move-Into the base
base: the enclosure that the object is unable to Move-Into

Examples:

  1. Fido was shut-out of his cage.
    object: Fido
    Comment: Fido is outside his cage and cannot get in.
    base: cage

Neighboring Concepts:

Similar to...But different because...
Be-Obstructed Be-Obstructed is more general, preventing all movements.
Be-Confined Be-Confined restricts the movement of the object out of a certain enclosure.
Be-Contained An object is contained in a container. It can Move-Out of it at any time.
Be-Restrained The object of a Be-Restrained state is unable to Move.
Be-Supported The object of a Be-Supported is prevented from Falling.
Be-Held has an agent that is allowed to Move the object.

Comment: An object can be put in a Be-Shut-Out state by a Shut-Out action, and can be taken out of this state by an Admit action.
